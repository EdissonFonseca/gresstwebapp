# Architecture Decision Records (ADR)

Lightweight ADRs for technical decisions that affect the project. One file per decision.

## Format

Each ADR is a markdown file with:

- **Title** — Short decision name
- **Status** — Proposed | Accepted | Deprecated | Superseded
- **Context** — What forces the decision
- **Decision** — What we decided
- **Consequences** — Trade-offs and follow-ups

## Naming

`NNNN-short-slug.md` (e.g. `0001-feature-driven-structure.md`). Number sequentially.
